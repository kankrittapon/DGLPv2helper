## 1. 🔄 AI Fallback System

**ปัญหา:** ถ้า AI Provider หลักพัง (rate limit, key หมดอายุ, server ล่ม) ต้องไปเปลี่ยนเอง

**แก้:** ลองตัวถัดไปอัตโนมัติ

```
ถาม Gemini → ❌ 429 Rate Limited
  ↓ ลอง Grok → ❌ 401 Invalid Key  
    ↓ ลอง Claude → ✅ สำเร็จ!
```

### สิ่งที่ต้องทำ
- [ ] เพิ่ม config `fallbackEnabled` (default: true)
- [ ] เพิ่ม config `fallbackOrder` (ลำดับ provider ที่จะลอง)
- [ ] แก้ `askAI()` ใน `popup.js` — loop ผ่าน provider ทีละตัว
- [ ] ข้าม provider ที่ไม่มี API Key
- [ ] แสดง log แต่ละ attempt ใน UI (❌/✅ + เหตุผล)

## 3. 🗄️ Supabase Backend

**ปัญหา:** ข้อมูลหายเมื่อลบ Extension หรือเปลี่ยนเครื่อง

**แก้:** เก็บทุกอย่างบน Supabase

### Database Schema

```sql
-- ตาราง quiz_attempts
CREATE TABLE quiz_attempts (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id TEXT NOT NULL,
  topic_name TEXT,
  course_id TEXT,
  section_id TEXT,
  total_questions INT,
  answers JSONB,                -- {"1":"ข","2":"ค","3":"ง",...}
  score INT,                    -- คะแนน (user ใส่เอง)
  ai_provider TEXT,             -- gemini/claude/grok/local/manual
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Index
CREATE INDEX idx_attempts_user ON quiz_attempts(user_id);
CREATE INDEX idx_attempts_topic ON quiz_attempts(course_id, section_id);
```

### Authentication
- ใช้ Anonymous UUID (สร้างอัตโนมัติ เก็บใน chrome.storage)
- ไม่ต้อง login — เปิดใช้งานได้เลย
- RLS ป้องกันคนอื่นเห็นข้อมูล

### ไฟล์ใหม่: `supabase.js`
```javascript
// ฟังก์ชันหลัก
getUserId()        // สร้าง/ดึง UUID
saveAttempt()      // บันทึก attempt ใหม่
getAttempts()      // ดึง attempts ตาม topic
updateScore()      // อัพเดทคะแนน
```

### Config เพิ่ม
```javascript
{
  supabaseUrl: '',      // Supabase Project URL
  supabaseKey: '',      // Supabase Anon Key
}
```
